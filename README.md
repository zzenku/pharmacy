💊 PharmaSub — Аптека по подписке

Интернет-аптека с подписочной моделью. Подписчики получают доступ к оптовым ценам на лекарства — экономия до 45% на каждой покупке.
---

📋 О проекте
PharmaSub — веб-платформа, позволяющая частным лицам покупать лекарства по оптовым ценам через подписку. Без подписки каталог доступен только для просмотра; оформив подписку за 990 ₽/месяц, пользователь получает доступ к корзине и ценам, недоступным в розничных аптеках.
Ключевые возможности

🔐 Система авторизации — три роли: гость, подписчик, администратор

💊 Каталог препаратов — 50+ наименований, фильтрация по категориям, поиск

💳 Двойная система цен — розничная и оптовая цена на каждый товар

🛒 Корзина и оформление заказов — доступно только подписчикам

📦 История заказов — в личном кабинете пользователя

🛡️ Панель администратора — управление товарами, пользователями и заказами

🔒 Безопасность — PDO с подготовленными запросами, защита от SQL-инъекций

🚀 Установка и запуск
Требования
XAMPP (PHP 8+, MySQL, Apache)
Браузер
Шаги
Клонируйте репозиторий в папку `htdocs` вашего XAMPP:
```bash
   git clone https://github.com/ваш-username/pharmasub.git
   cd xampp/htdocs/pharmasub
   ```
Запустите XAMPP — включите модули Apache и MySQL.
Создайте базу данных через phpMyAdmin:
Создайте БД с именем `pharmacy`
Импортируйте файл `pharmacy.sql` (Импорт → выбрать файл)
Настройте подключение — откройте `config.php` и при необходимости скорректируйте параметры:
```php
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'pharmacy');
   define('DB_USER', 'root');
   define('DB_PASS', '');
   ```
Откройте в браузере:
```
   http://localhost/pharmasub/
   ```

🗄 Схема базы данных
База данных `pharmacy` содержит 6 таблиц:
```
users          — пользователи, роли, статус подписки
products       — товары (price_retail + price_wholesale)
categories     — категории препаратов
cart           — корзина (связь user ↔ product)
orders         — заказы с адресом и статусом
order_items    — позиции заказа (связь order ↔ product)
```
Ключевое решение: каждый товар хранит две цены — `price_retail` и `price_wholesale`. При добавлении в корзину система автоматически применяет нужную цену на основе роли пользователя.
